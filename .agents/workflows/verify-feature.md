# Verify Feature Workflow

This workflow is designed to ensure a newly developed feature is thoroughly tested, fully implemented, and issue-free by coordinating a team of subagents.

1.  **Test Planning and Execution**:
    -   Invoke subagents to write comprehensive unit and integration tests for the new feature.
    -   Execute the test suite and collect results.
    -   Identify any failing tests or uncovered edge cases.

2.  **Issue Resolution**:
    -   Delegate the failing tests and identified issues to appropriate subagents.
    -   Iterate on fixing the bugs until all tests pass and no further issues are found.

3.  **Feature Assessment**:
    -   Compare the implemented feature against the original requirements and specifications.
    -   Identify any missing components, user interfaces, or backend logic.

4.  **Implementation of Missing Parts**:
    -   Lead a team of subagents to implement any missing functionality identified in the assessment phase.
    -   Ensure the new additions integrate seamlessly with the existing codebase.
    -   Repeat steps 1 and 2 for the newly implemented parts.

5.  **Final Verification**:
    -   Perform a comprehensive final test to ensure the entire feature works harmoniously.
    -   Document the successful verification and any subsequent implementation details in an artifact.
