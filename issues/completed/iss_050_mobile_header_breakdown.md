# Bug: Mobile Header Layout Breakdown (375px)

## Description
When viewing the dashboard on a 375px width (common mobile viewport), the header layout breaks. The logo and profile toggle are oversized and stacked vertically, consuming excessive vertical space and misaligning with the rest of the UI.

## Steps to Reproduce
1. Log in to the application.
2. Navigate to the Dashboard.
3. Resize the browser window to 375px width.
4. Observe the top header section.

## Expected Result
A compact, responsive header where the logo and profile toggle are aligned horizontally or integrated into a single row to save vertical space.

## Actual Result
Elements are oversized, stacked vertically, and misaligned.

## Impact
**Major (Visual/UX).** Seriously degrades the mobile experience and makes the app look unpolished on small screens.
