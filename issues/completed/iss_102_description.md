# Coding Activity Heatmap Month Labels Overlap on Mobile

## Description
In the Student Report Card view (Parent portal), the "Coding Activity" heatmap's month labels (Jan, Feb, Mar, etc.) overlap each other, making them completely unreadable on mobile screens.

## Steps to Reproduce
1. Log in as a Parent.
2. Navigate to a connected child's Report Card on a mobile device (width 390px).
3. Scroll down to the "Coding Activity" section.
4. Observe the month labels at the top of the heatmap.

## Expected Result
The month labels should be spaced out appropriately, or fewer labels should be shown (e.g., every other month), or the heatmap should be horizontally scrollable to prevent overlapping text.

## Actual Result
The month text strings overlap each other severely (e.g. "JudulAugSepOctNov..."), making them illegible.

## Impact
Low - The actual activity blocks are still visible, but the labels look broken.
