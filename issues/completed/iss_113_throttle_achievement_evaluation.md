# Issue: Throttle Achievement Evaluation

## Description
The implementation of `evaluate_user()` in `achievement_engine.py` is called synchronously on every visit to the `/achievements` page. This engine performs dozens of aggregate `COUNT(*)` and `SELECT *` queries across large tables (SessionLogs, ChallengeLogs, Messages) to check if a user has earned any of the ~50 achievements.

For 50 users, this causes a noticeable bottleneck on the achievement page and places heavy load on the SQLite file.

## Proposed Solution
Implement a throttling mechanism to prevent constant re-evaluation.
1.  **Session Throttling**: Only run the full evaluation once per hour per user, or once per login session.
2.  **Trigger-based Evaluation**: (Optional) Evaluate specifically when an action occurs (e.g. after sending a message), rather than every page refresh.
3.  **Optimize Count Queries**: Update the achievement logic to use more efficient SQLAlchemy queries or cached counters.

## Acceptance Criteria
- [ ] Achievement page load time is reduced.
- [ ] Number of database queries generated by `get_achievements_json` is significantly reduced.
- [ ] New achievements are still eventually granted (within a reasonable delay).

## Priority
Medium (Quick Win)
